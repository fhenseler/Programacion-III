JSON WEB TOKEN: crear en el login solamente (o cuando queda poco tiempo)
ENTENDER EL PROCESO DEL TOKEN (CUANDO SE CREA Y CUANDO SE PASA) 

POSTMAN: LOGIN (USUARIO PASS), NOS DA EL TOKEN
OTRA SOLAPA DE POSTMAN PARA HACER EL ALTA USANDO EL TOKEN (COPIADO Y PEGADO) SIN BASE DE DATOS
IF EN EL MIDDLEWARE PARA VER SI EL TOKEN ES VALIDO
EL TOKEN SE RECIBE Y SE ENVIA POR HEADER (VER EN EL EJEMPLO PPT)
